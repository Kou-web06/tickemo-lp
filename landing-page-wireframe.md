# Tickemo LP ワイヤー案（SP/PC）

## 1. SPワイヤー（優先）

### 1) Hero
- 上: ロゴ
- 中: 見出し + サブコピー
- 下: 主CTA
- 背景: ダーク + ぼかし + 実機1〜2枚

### 2) 価値訴求3カード
- 縦積みカード x3
- 各カード: アイコン / 見出し / 1〜2行本文

### 3) 機能ハイライト
- セクション見出し
- 機能ブロックを縦積み（画像→テキスト）x6

### 4) スクリーンショットギャラリー
- 横スクロール
- 5〜7枚

### 5) 利用シーン
- 見出し
- 箇条書き3点

### 6) FAQ
- アコーディオン3項目

### 7) 最終CTA
- 見出し
- 主CTA

### 8) フッター
- 規約 / プライバシー / 問い合わせ
- Apple Musicクレジット

## 2. PCワイヤー

### ファーストビュー
- 左: 見出し・本文・CTA
- 右: 実機モック（2台重ね）

### 価値訴求
- 3カラムカード

### 機能ハイライト
- 2カラム交互（画像左テキスト右 / 反転）

### ギャラリー
- 3〜4列グリッド + 軽いホバー

### 利用シーン + FAQ
- 左: 利用シーン
- 右: FAQアコーディオン

### 最終CTA + フッター
- 横幅いっぱいのCTA帯
- 下にフッターリンク

## 3. UIパーツ仕様
- 主CTAボタン高: SP 52px / PC 56px
- セクション余白: SP 56〜72px / PC 88〜120px
- カード角丸: 16〜24px
- ベース背景: #0F0F10 近辺
- テキスト最大幅: PCで 640px 目安

## 4. 画像配置ルール
- スクリーンショットは端末フレーム付きまたは同一比率で統一
- 画像内文字が読めるよう暗背景に対して境界を確保
- Hero画像は情報過多にしない（UI要素は2画面まで）

## 5. モバイル体験上の注意
- CTAは初回表示領域内に1つ必須
- ヒーロー直後に価値訴求3カードを置く
- 長文禁止（1ブロック2〜3行上限）
